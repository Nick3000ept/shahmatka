# СБ3 Шахматка — Чеклист проверок

Этот файл описывает что и как проверять в коде: для самоконтроля, code review и понимания логики работы панели.

---

## 1. Данные не перезаписываются без изменений

**Где проверять:** `script.gs` → `saveOneRow()` и `saveAll()`

**Как устроено:**
- В обоих методах есть флаг `anyChange = false`
- Поля `status`, `pct`, `dateEnd`, `org`, `comment` устанавливают `anyChange = true` только при наличии значения
- `DATE_CHG` обновляется только если `anyChange === true`
- `saveOneRow` завершается без записи в таблицу если `!anyChange`
- `saveAll` добавляет строку в `changedIndices` только если `anyChange === true` → только эти строки пишутся обратно

**На фронтенде:**
- 5 флагов `_statusTouched`, `_pctTouched`, `_dateTouched`, `_commentTouched`, `_contractorTouched`
- Флаги устанавливаются только при реальном взаимодействии (клик, ввод, изменение)
- В `savePanel()` поля пишутся в `MOD` только если соответствующий флаг `true`
- Если ни один флаг не сработал — `MOD[rid]` удаляется, строка не попадает в очередь на сохранение

**Что проверить:**
```
□ В saveOneRow есть строка: if (!anyChange) return;
□ В saveAll строка добавляется в changedIndices только если anyChange
□ В savePanel каждое поле обёрнуто в if(_touched)
□ После if(...touched) есть: if(Object.keys(MOD[rid]).length===0) delete MOD[rid];
□ Все 5 флагов сбрасываются в openPanel() перед открытием карточки
```

## 2. Строки не удаляются

**Где проверять:** `script.gs` — весь файл

**Как устроено:**
- Нет ни одного вызова `sheet.deleteRow()`, `sheet.deleteRows()`, `sheet.clearContent()`
- `saveOneRow` только читает и перезаписывает ячейки G–N одной строки
- `saveAll` читает весь лист G–N, модифицирует только изменённые строки, пишет обратно
- Ни один эндпоинт не принимает `action=delete` или аналог

**Что проверить:**
```
□ Grep по слову "deleteRow" в script.gs → результатов нет
□ Grep по слову "clearContent" в script.gs → результатов нет
□ В doPost обрабатываются только action: saveRow и saveAll
□ В doGet обрабатываются только: getRows, getContractors, checkPassword, clearCache, ping
```

## 3. У каждой изменённой строки сохраняется дата и автор

**Где проверять:** `script.gs` → `saveOneRow()` и `saveAll()`

**Как устроено:**
- Столбец M (индекс 13, `C.DATE_CHG`) = дата изменения в формате `dd.MM.yyyy`
- Столбец N (индекс 14, `C.AUTHOR`) = имя автора из поля "Ваше имя" в шапке
- Оба поля обновляются только если `anyChange === true`
- На фронтенде: после успешного сохранения `batchSave()` обновляет `row.dateChg` и `row.author` в локальном массиве `ROWS`
- В карточке задачи отображается: "Изменено: дата · Автор: имя"

**Что проверить:**
```
□ В saveOneRow: if (anyChange) rowValues[C.DATE_CHG - off] = nowStr;
□ В saveOneRow: if (data.author) rowValues[C.AUTHOR - off] = data.author;
□ В saveAll: аналогичные строки внутри if (anyChange)
□ В batchSave (index.html): после успеха row.dateChg и row.author обновляются
□ В openPanel: po-meta заполняется из row.dateChg и row.author
```

## 4. Поля администратора скрыты для остальных

**Где проверять:** `index.html`

**Что скрыто для не-администраторов:**
| Элемент | ID | Описание |
|---|---|---|
| Фильтр по дате | `#fdate-inp` + иконка | Показывает работы до указанной даты |
| Подрядчик в карточке | `#fg-contractor` | Назначение подрядчика на строку |
| Подрядчик в мультивыборе | `#ms-contractor` + метка + разделитель | Массовое назначение подрядчика |

**Как устроено:**
- CSS: `.admin-only { display: none !important; }` — скрыт по умолчанию
- `applyAdminUI()` снимает класс `.admin-only` если `IS_ADMIN === true`
- `IS_ADMIN` устанавливается из `sessionStorage.getItem('sb3_admin') === '1'` при загрузке
- После успешного ввода пароля: `IS_ADMIN = true` + `applyAdminUI()` + запись в sessionStorage

**Что проверить:**
```
□ CSS содержит: .admin-only { display: none !important; }
□ Элементы fdate-inp, fg-contractor, ms-contractor имеют класс admin-only в HTML
□ applyAdminUI() вызывается в window.onload
□ applyAdminUI() вызывается после успешного входа (IS_ADMIN=true)
□ IS_ADMIN читается из sessionStorage при старте (не localStorage — сессия, не постоянно)
```

## 5. Целостность диапазона записи (столбцы A–F защищены)

**Где проверять:** `script.gs` → `saveOneRow()` и `saveAll()`

**Как устроено:**
- Столбцы A–F содержат dropdown-валидацию Google Sheets
- Запись туда вызывает: `Exception: Данные не соответствуют правилам проверки`
- `EDIT_START = 7` (столбец G), `EDIT_COLS = 8` (диапазон G:N)
- Запись: `sheet.getRange(row, 7, 1, 8)` — никогда не затрагивает A–F
- Резервная защита: `try { запись G:N } catch(e) { запись H:N }` — если G (org) тоже валидируется

**Что проверить:**
```
□ В saveOneRow: getRange(targetRowSheet, 7, 1, 8) — первый числовой аргумент = 7
□ В saveAll: getRange(idx+2, 7, 1, 8) при записи изменённых строк
□ Есть try/catch: при ошибке fallback на getRange(..., 8, 1, 7) — пишем H:N
□ Столбцы O+ никогда не читаются для записи (только P=baseDate, Q=currentDate на чтение)
```

## 6. Конкурентные записи (несколько пользователей одновременно)

**Где проверять:** `script.gs` → `saveAll()`

**Как устроено:**
- `saveAll` использует `LockService.getScriptLock()` — только один процесс пишет в таблицу
- Второй пользователь ждёт до 15 секунд (`waitLock(15000)`)
- Чтение данных происходит **внутри лока** → читается уже обновлённое состояние
- Лок освобождается в блоке `finally` → гарантированно, даже при ошибке
- `saveRow` (одиночная строка) работает без лока — параллельные записи в разные строки безопасны

**Что проверить:**
```
□ В saveAll: lock = LockService.getScriptLock()
□ lock.waitLock(15000) вызывается до чтения данных
□ Чтение idValues и allValues — внутри try{} после waitLock
□ lock.releaseLock() — в блоке finally{}
□ saveOneRow НЕ использует LockService (параллельные одиночные — не конфликтуют)
```

## 7. Роли и доступ к редактированию

**Где проверять:** `index.html` → `cellClick()`, `toggleRow()`, `toggleCol()`

**Как устроено:**

| Роль | Как определяется | Может редактировать |
|---|---|---|
| Администратор | Пароль `adminACCB3` → `IS_ADMIN=true` | Все строки |
| Подрядчик | URL `?contractor=Название` → `CONTRACTOR` | Только строки где `org === CONTRACTOR` |
| Просмотр | Без пароля и без параметра | Нельзя, просит пароль |

- Ячейки не-своих строк получают класс `.not-mine` (opacity 0.35) — визуальная индикация
- При мультивыборе чужие строки пропускаются + показывается тост "Пропущены работы других подрядчиков"
- Подрядчик автоматически видит только свои работы через авто-установку фильтра ОРГАНИЗАЦИЯ

**Что проверить:**
```
□ cellClick: содержит проверку IS_ADMIN и CONTRACTOR перед открытием редактирования
□ toggleRow: пропускает строки где org !== CONTRACTOR
□ toggleCol: пропускает строки где org !== CONTRACTOR
□ isNotMine = CONTRACTOR && !IS_ADMIN && row.org !== CONTRACTOR — используется в render()
□ buildFilters: при CONTRACTOR && !IS_ADMIN → os.value = CONTRACTOR, FO = CONTRACTOR
```

## 8. Производительность рендера

**Где проверять:** `index.html` → `render()` ~строка 999

**Как устроено:**
- `COL_KEYS` строится за **один O(n) проход** по строкам (не вложенные циклы)
- Фильтрация `ROWS.filter(...)` — один проход
- HTML строится как строка (`h += ...`) и вставляется один раз через `innerHTML`
- `renderDeferred()` — дебаунс 150мс для поиска по работам, не вызывает render() на каждый символ

**Что проверить:**
```
□ В render(): COL_KEYS строится в одном forEach по rows, без вложенных .some()/.filter()
□ Нет вложенных циклов rows × rows (O(n²))
□ HTML собирается в переменную h и вставляется одним board.innerHTML = h
□ renderDeferred использует setTimeout для дебаунса
```

## 9. Безопасность данных при сохранении

**Где проверять:** `index.html` → `saveAll()` и `batchSave()`

**Как устроено:**
- Перед сохранением проверяется наличие `AUTHOR` — без имени сохранение блокируется
- Отправляются **только поля из MOD** — лишние данные не попадают на сервер
- GAS принимает только известные поля (`status`, `pct`, `dateEnd`, `org`, `comment`, `author`, `rowId`)
- GAS не исполняет произвольный код из запроса

**Что проверить:**
```
□ saveAll(): if(!AUTHOR) { toast(...); return; }
□ batchSave(): payload строится только из MOD[rid] — нет лишних полей
□ GAS saveOneRow/saveAll: обрабатываются только data.status, data.pct, data.dateEnd, data.org, data.comment, data.author
□ Нет eval(), Function(), динамического выполнения строк из запроса
```

## 10. Дополнительные рекомендуемые проверки

### 10.1 Структура листа не нарушена
```
□ Первая строка листа СБ3_ОБЩАЯ — заголовки (строка 1), данные с строки 2
□ lastRow = sheet.getLastRow() — не хардкодится
□ При пустом листе (lastRow < 2) — ранний выход без ошибки
```

### 10.2 Обработка ошибок
```
□ batchSave: при ошибке fetch показывает toast с текстом ошибки
□ saveOneRow в GAS: обёрнут в try/catch в doPost
□ Ошибки GAS возвращаются в { error: '...' } — не проглатываются молча
□ Таймаут LockService (15s): если истёк — пользователь видит ошибку, не зависает
```

### 10.3 Целостность фронтенда после сохранения
```
□ После успешного batchSave: MOD[rid] удаляется (delete MOD[rid])
□ После сохранения: локальный ROWS[row] обновляется — UI показывает актуальные данные без перезагрузки
□ После сохранения: render() вызывается — таблица перерисовывается
```

### 10.4 Пресеты и localStorage
```
□ Пресеты хранятся в localStorage под ключом sb3_presets
□ При парсинге пресетов есть try/catch — битый JSON не ломает страницу
□ Пресет содержит только фильтры — не содержит данных таблицы
□ Удаление пресета: delete PRESETS[name] + localStorage.setItem(...)
```

### 10.5 Что НЕ должно появиться в коде (красные флаги)
```
□ sheet.deleteRow() или sheet.deleteRows() — в script.gs отсутствует
□ sheet.clearContent() или sheet.clear() — отсутствует
□ Запись в столбцы A–F (getRange с начальным столбцом < 7) — отсутствует
□ Запись в столбцы O+ (getRange с диапазоном > 14) — отсутствует
□ eval() или new Function() в index.html — отсутствует
□ action=delete в doGet/doPost — отсутствует
```

## Как использовать этот файл

**При каждом значимом изменении кода** пройдись по чеклисту раздела, который ты менял. Особенно важны разделы 1, 2, 5.

**При добавлении нового поля данных** в таблицу:
1. Добавь константу в `C` объект в `script.gs`
2. Добавь в `rows.push()` в `getRows()`
3. В `saveOneRow` и `saveAll` добавь запись только если `C.X` входит в диапазон G–N
4. Добавь `_touched` флаг на фронтенде
5. Обнови этот файл

**При изменении ролей/доступа:**
- Проверь раздел 7
- Убедись что новые ограничения применяются в `cellClick`, `toggleRow`, `toggleCol`
- UI-элементы для администратора — добавь класс `admin-only`
